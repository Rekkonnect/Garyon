# Garyon Documentation Sources

This directory contains the DocFX configuration, build scripts, generated artifacts, and the Markdown guides that back the Garyon documentation site.

## Layout

- `docfx.json` is the DocFX entrypoint.
- `toc.yml` defines the top-level site navigation.
- `guides/` contains the user guides.
- `api/` and `_site/` are generated by DocFX.

## Build

From the repository root:

```powershell
.\docs\build-docs.ps1
.\docs\build-docs.ps1 -Serve
```

```bash
chmod +x docs/build-docs.sh
./docs/build-docs.sh
./docs/build-docs.sh --serve
```

Or run DocFX directly:

```bash
docfx docs/docfx.json
docfx docs/docfx.json --serve
docfx metadata docs/docfx.json
docfx build docs/docfx.json
```

## Writing Guides

Guides live in `guides/` and should:

- stay practical and example-driven
- link to related guides and API pages
- be added to `guides/toc.yml`

Example links from files in this directory:

- to another guide: `[Task Handling](guides/task-handling.md)`
- to the API reference: `[IEnumerableExtensions](api/Garyon.Extensions.IEnumerableExtensions.yml)`

## Troubleshooting

- Ensure DocFX is installed: `dotnet tool install -g docfx`
- Build the library if metadata generation fails.
- Run `docfx docs/docfx.json` to catch broken links before committing.
